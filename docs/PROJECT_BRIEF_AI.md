# DataLens 项目简述（产品视角，AI可读）

## 1) 文档用途

本文件用于让任意 AI 或新成员快速理解：

- 这个项目解决什么问题
- 用户是谁、核心价值是什么
- 当前功能边界与非目标
- 业务流程如何从“提问”走到“可执行结果”

## 2) 一句话定义

DataLens 是一个「数据表智能理解 + 自然语言转 SQL + 只读执行预览」的轻量分析系统。

## 3) 术语表（标准定义）

- `数据源`：可连接的数据系统实例（当前支持 MySQL / ClickHouse）。
- `数据库`：数据源中的逻辑库。
- `数据表`：数据库中的表对象。
- `字段语义`：字段在业务中的含义（如订单金额、用户ID、下单时间）。
- `Profiling`：对样本数据做统计分析（空值率、去重数、top值等）。
- `RAG`：检索增强生成，先检索历史语义，再让模型生成 SQL。
- `Copilot`：面向自然语言问答的 SQL 助手页面。

## 4) 目标用户与核心痛点

目标用户：

- 数据分析师
- 数仓/数据开发
- 需要接手陌生数据表的业务分析角色

核心痛点：

- 不知道陌生表字段含义
- 不清楚表可以支持哪些分析场景
- 写 SQL 需要频繁翻文档/问同事

## 5) 核心目标（当前版本）

- 快速接入数据源并浏览库表
- 自动生成字段语义与数据质量画像
- 自动给出表级用途总结和可分析场景
- 用自然语言生成 SQL，并返回执行结果预览

## 6) 用户可见功能

- 数据源管理：新增、编辑、删除、测试连接
- 数据目录浏览：按数据源/数据库查看表清单
- 分析触发：支持整源、整库、单表分析
- 表详情页：展示字段语义、质量指标、表总结
- 业务域管理：维护业务描述并选择相关库表
- Copilot 对话：问问题 -> 生成 SQL -> 展示解释与执行结果

## 7) 端到端业务流程（AI建议按步骤理解）

1. 用户录入数据源并测试连接  
2. 用户选择分析范围（表/库/数据源）  
3. 系统抽取结构并做字段统计  
4. 系统调用 LLM 生成字段语义与表总结  
5. 系统写入结构化结果与向量索引  
6. 用户在 Copilot 输入自然语言问题  
7. 系统检索上下文并生成 SQL  
8. 系统执行只读 SQL，返回结果预览  

## 8) 设计原则（产品约束）

- `MVP优先`：先打通核心闭环，不追求大而全
- `可解释优先`：结果必须附带语义、总结和解释
- `可退化运行`：无模型 Key 时也能联调主流程
- `安全默认`：仅允许只读查询执行

## 9) 当前边界（In Scope）

- 支持 MySQL / ClickHouse
- 支持表语义理解 + SQL 生成 + SQL 只读执行预览
- 支持业务域定义与库表选择
- 支持单用户本地/轻量部署场景

## 10) 非目标（Out of Scope）

- 数据血缘与数据治理平台能力
- 复杂权限体系、多租户、审计合规体系
- 团队协作工作流（审批、评审、共享空间）
- 高级指标平台与统一口径引擎

## 11) 关键输入/输出（便于AI调用）

关键输入：

- 数据源连接信息
- 用户自然语言问题
- 可选的 `table_id`（限定提问范围）

关键输出：

- 字段级语义与质量指标
- 表级总结（用途、场景、关键字段、告警）
- SQL + explanation + 执行结果（列与行）

## 12) 现状与初始设计差异

相对初始设计，当前实现新增/增强：

- 新增“业务域”管理（业务描述 + 库表映射）
- Copilot 已支持 SQL 执行与结果返回（不再只生成 SQL）
- 支持无 Key 的规则兜底与本地向量兜底，方便开发联调
