## 1. 系统概述

DataLens 是一个 **LLM 驱动的企业级语义知识图谱平台**，核心工作流为：

```
数据源接入 → 文档索引 → LLM 语义抽取 → 本体知识图谱 → Copilot 智能问答
```

五层语义资产模型：

| 层级 | 英文 | 说明 |
|------|------|------|
| 词汇层 | Vocabulary | 业务术语定义、同义词 |
| 实体概念层 | Entity-Concept | 指标、维度、物理表、业务规则等核心实体 |
| 关系层 | Relation | 实体间的语义关系（依赖、派生、聚合等） |
| 属性层 | Attribute | 实体的属性断言（公式、口径、置信度等） |
| 规则层 | Rule | 业务规则与约束表达式 |

## 2. 初始配置

### 2.1 配置 LLM 模型连接

DataLens 依赖大语言模型进行语义抽取和智能问答，必须先配置。

1. 打开侧栏，点击 **「设置」**
2. 在设置页面选择 **「模型与连接」** 标签页
3. 点击 **「添加连接」**，填写：
   - **厂商**：从下拉列表中选择（支持 OpenAI、Azure、Anthropic、Google、DeepSeek、Moonshot、阿里百炼、字节豆包、百度千帆、智谱 AI、OpenRouter、Mistral、Groq、Cohere、SiliconFlow、Amazon Bedrock、腾讯云、自定义等 18+ 厂商）
   - **连接名称**：自定义，如 `生产环境-GPT-4o`
   - **Base URL / Endpoint**：API 端点地址（厂商默认值自动填充）
   - **API Key**：服务商提供的密钥
   - **模型 ID**：如 `gpt-4o`、`deepseek-chat` 等
4. 点击 **「测试连接」** 验证连通性
5. 保存后，连接卡片出现在列表中

### 2.2 配置语义策略

1. 切换到 **「语义策略」** 标签页
2. 选择用于本体抽取（表分析、知识图谱构建）的模型：
   - **自动**：系统自动选择可用模型
   - 或手动指定某个已配置的连接
3. 页面底部显示当前解析到的模型名称、厂商和模型 ID

### 2.3 配置 Copilot 聊天模型

1. 切换到 **「Copilot 聊天模型」** 标签页
2. 选择用于智能问答的模型（可与语义策略使用不同模型）

### 2.4 配置界面主题

1. 切换到 **「界面」** 标签页
2. 选择 **浅色 / 深色 / 跟随系统**

### 2.5 配置 API 数据源（可选）

如需从 Notion、Confluence、飞书等外部平台导入文档：

1. 切换到 **「API 数据源」** 标签页
2. 选择对应平台，填写集成令牌和应用凭证
3. 点击 **「测试连接」** 验证

## 3. 数据源管理

DataLens 支持连接关系型数据库并自动分析其 Schema。

### 3.1 添加数据库连接

1. 点击侧栏 **「数据源」**，进入数据源列表页
2. 点击 **「添加数据源」** 按钮
3. 第一步 — 选择数据库类型：
   - MySQL / PostgreSQL / ClickHouse / SQLite 等
4. 第二步 — 填写连接信息：
   - **名称**：自定义，如 `生产数据仓库`
   - **主机**：数据库 IP 或域名
   - **端口**：数据库端口号
   - **数据库名**：默认连接的数据库
   - **用户名 / 密码**：数据库凭据
5. 点击 **「测试连接」** 确认连通
6. 点击 **「保存」**

### 3.2 分析数据库 Schema

1. 在数据源列表中点击已连接的数据源，进入详情页
2. 页面列出该数据源下的所有数据库
3. 点击 **「分析数据源」** 按钮，系统将调用 LLM 自动分析所有表的 Schema：
   - 识别每列的业务含义和语义类型
   - 生成表的业务摘要、数据定位、核心指标推测、使用建议
4. 也可进入某个数据库，逐表点击 **「分析」** 进行分析

### 3.3 查看表分析结果

1. 进入数据源 → 数据库 → 点击某张表
2. 表详情页展示：
   - **AI 结构化摘要**：业务描述、数据定位、核心指标、使用建议、风险边界
   - **列清单**：每列的语义类型、业务描述
   - **关联知识库**：该表被哪些知识库引用
   - **Copilot 校验面板**：快速提问验证数据内容

## 4. 业务域管理

业务域（Business Domain）是 DataLens 的顶层组织单元，用于划分不同的业务上下文。

### 4.1 创建业务域

1. 点击侧栏 **「设置」** → **「业务域管理」** 标签页
2. 点击 **「新增业务域」**
3. 填写 **名称**（如「电商交易」「财务核算」）和 **描述**
4. 点击保存

### 4.2 关联知识库和表

1. 在侧栏顶部下拉框选择当前业务域
2. 进入业务域详情页（点击侧栏业务域名或 `/business-domains/[id]`）
3. 可操作：
   - **编辑业务描述**：富文本编辑器
   - **关联知识库**：勾选已有知识库
   - **批量添加表**：从数据源目录树中选择表加入该业务域

### 4.3 切换业务域

侧栏顶部下拉框切换。切换后，所有页面（知识库、语义资产、Copilot）都将限定在当前业务域的上下文中。

## 5. 知识库与数据导入

知识库（Knowledge Base）是文档和本体的容器，一个业务域可包含多个知识库。

### 5.1 创建知识库

1. 点击侧栏 **「知识库」**，进入知识库列表页
2. 点击 **「新建知识库」** 按钮
3. 在弹出的对话框中填写：
   - **名称**（必填）：如「指标体系口径文档」
   - **描述**（可选）：说明知识库的范围和用途
4. 点击确认，跳转到知识库详情页

### 5.2 导入数据 —— 上传文件

1. 在知识库详情页，点击 **「导入数据」** 按钮
2. 在导入向导中，第一步选择连接器为 **「上传文件」**
3. 第二步，拖拽或点击上传文件（支持 `.md` / `.txt` / `.csv` / `.sql` / `.pdf` 等格式）
4. 文件上传后自动进入清洗流水线：
   ```
   提取文本 → 清洗规范化 → 分块 → 语义结构化 → 向量嵌入 → 索引存储
   ```

### 5.3 导入数据 —— Git 仓库

1. 在知识库详情页，点击 **「导入数据」** → 选择 **「Git 仓库」**
2. 填写 Git 源配置：
   - **名称**、**提供商**（GitHub / GitLab）
   - **API 基础 URL**、**Owner**、**仓库名**
   - **分支**（默认 `main`）、**路径前缀**（可选，只索引子目录）
   - **Token**：用于 API 认证的 Personal Access Token
   - **文件匹配**（include globs）：默认 `*.sql,*.py,*.yml,*.yaml,*.hql`
   - **提取策略**：
     - `data_warehouse`：跳过 `.ts/.tsx/.jsx` 前端文件
     - `mixed`：不做过滤
   - **启用文档索引**：是否创建可搜索的文本块（仅解析血缘时不勾选）
   - **Crontab 表达式**（可选）：定时自动同步，如 `0 3 * * *` 每天凌晨3点
3. 保存后，点击源卡片上的 **「Sync Now」** 触发首次同步
4. 文件同步后在源详情页可查看文件树、代码内容和文档块质量分

### 5.4 导入数据 —— 数据库 Schema

1. 点击 **「导入数据」** → 选择 **「数据库连接器」**
2. 系统列出已配置的数据源，选择目标数据源
3. 勾选要导入的数据库（一个或多个）
4. 点击确认 → 系统导入所选数据库的表结构
5. 导入完成后，在源卡片上点击 **「语义清洗」** → 将物理表结构写入本体知识图谱

### 5.5 导入数据 —— 手动录入

1. 点击 **「导入数据」** → 选择 **「手动录入」**
2. 填写 **标题** 和 **正文**
3. 保存后自动进入清洗流水线

### 5.6 导入数据 —— TTL 本体导入

1. 点击 **「导入数据」** → 选择 **「TTL 导入」**
2. 粘贴或输入 Turtle 格式的 RDF 数据
3. 提交后经过 SHACL 校验写入知识图谱

### 5.7 查看导入状态

在知识库详情页的 **「证据包注册表」**（Evidence Package List）区域：

- 每个导入来源显示一张卡片，标注连接器类型
- 状态指示：
  - ⏳ `pending` — 等待处理
  - 🔄 `running` — 正在清洗（显示当前阶段名称）
  - ✅ `completed` — 完成
  - ❌ `failed` — 失败（可查看错误详情并重试）
- 前端每几秒自动轮询刷新状态

## 6. 本体抽取与语义清洗

导入数据后，DataLens 自动运行**本体抽取流水线**，将非结构化文本转化为知识图谱三元组。

### 6.1 抽取流水线触发方式

- **文件上传 / Git 同步**：自动触发
- **数据库 Schema 导入**：需手动点击源卡片上的 **「语义清洗」** 按钮
- **手动触发**：在知识库详情页，点击对应源的 **「语义清洗」**

### 6.2 抽取流水线步骤

流水线并行执行以下抽取任务：

| 步骤 | 功能 | 输出 | 对应理论标准 |
|------|------|------|-------------|
| 术语抽取 | 识别业务术语及其定义 | `BusinessTerm` 实体 | ① 概念词典/业务词汇表 |
| 指标口径抽取 | 识别指标、公式、口径 | `Metric` 实体 | ⑥ ABox 实例 — 指标 |
| 维度抽取 | 识别分析维度及类型 | `Dimension` 实体 | ⑥ ABox 实例 — 维度 |
| 规则抽取 | 识别校验/派生/约束规则 | `BusinessRule` 实体 | ⑤a OWL 公理 / ⑤b SHACL 约束 / ⑤c SWRL 规则（按 `ruleType` 字段区分） |
| 关系抽取 | 识别实体间的语义关系 | `dependsOn` / `derivedFrom` / `relatedTo` 等 | ③ 关系定义（OWL ObjectProperty） |
| 属性值提取 | 提取实体属性字面量 | `dl:formula` / `dl:caliber` / `dl:dimensionType` 等 | ④ 属性定义（OWL DatatypeProperty）— 嵌入各实体抽取步骤产出 |
| 层级构建 | 构建概念父子层级 | `skos:broader` / `skos:narrower` | ② 类层次结构 |
| 血缘抽取 | 解析 SQL/Python 中的数据流转 | `LineageAssertion` 实体 | ③ 关系定义 — `transformsFrom`（TransitiveProperty） |
| Join 关系抽取 | 识别表间 Join 条件 | `joinableWith` 关系 | ③ 关系定义 — `joinableWith`（SymmetricProperty） |
| 跨库等价检测 | 检测跨知识库同义术语 | `skos:exactMatch` / `skos:closeMatch` | ⑦ 对齐与可视化 |
| 跨域本体对齐 | 检测跨业务域相似概念 | 跨域 `skos:closeMatch` 建议 | ⑦ 对齐与可视化 |

### 6.3 流水线后处理

以下后处理步骤对应理论标准产出物 ⑦ 对齐与可视化。

抽取完成后，系统自动执行：

1. **交叉实体一致性校验**（P0）：检查引用完整性 — 所有 `dependsOn`、`derivedFrom`、`computedFromTable` 等引用目标必须存在
2. **跨知识库等价检测**（P2）：使用 SBERT 语义向量模型检测新术语与已有知识库中术语的等价关系，自动添加 `skos:exactMatch`（相似度 ≥ 0.92）或 `skos:closeMatch`（相似度 ≥ 0.82）
3. **跨领域本体对齐**（P3）：检测不同业务域中语义相近的概念，建议跨域映射
4. **实体版本追踪**（P3）：对比新旧实体状态，检测标签/定义/公式/关系变更，自动生成版本号和变更记录

### 6.4 查看流水线进度

在知识库详情页的源卡片上可看到实时状态。也可以在 **「建模与质量」** 页面查看流水线运行的详细步骤和进度。

### 6.5 断点续跑

如果流水线因超时或异常中断，下次触发时会自动识别已完成步骤，从断点继续，避免重复调用 LLM。

## 7. 语义资产浏览

### 7.1 全局语义资产页

1. 点击侧栏 **「语义资产」**（或在顶部选择业务域后自动定位）
2. 页面顶部显示五层资产的总计数量
3. 使用标签页切换浏览层级：
   - **词汇**：术语名称、定义、状态、IRI
   - **实体概念**：实体名称、类型、层级邻居、IRI
   - **关系**：主体-谓词-客体三元组
   - **属性**：主体-属性-值断言
   - **规则**：规则名称、口径/表达式、公式、状态
   - **维度**：维度名称、类型（时间/地理/分类/层级）
4. 每个层级支持：
   - **搜索过滤**：输入关键词实时筛选
   - **分页浏览**：默认每页 20 条（规则层 10 条）
   - **按知识库过滤**：下拉选择特定知识库
   - **IRI 面包屑导航**：缩短显示的 IRI 便于阅读

### 7.2 查看溯源信息

1. 在任意资产列表中，点击某一项
2. 弹出 **溯源模态框**，展示：
   - 该实体的来源知识库
   - 来源文档或 Git 文件路径
   - 原始文本证据片段
   - LLM 抽取的置信度

### 7.3 知识库级别的建模结果

1. 进入知识库详情页，点击 **「建模与质量」** 按钮
2. 页面分为两个子标签页：
   - **「五层结果」**：展示该知识库的语义抽取成果，按五层分类，含数量统计
   - **「质量与隔离」**：SHACL 校验通过率、隔离区项数、置信度分布图

## 8. 建模与质量治理

### 8.1 查看 SHACL 校验结果

1. 进入知识库 → **「建模与质量」** → **「质量与隔离」** 标签页
2. **SHACL 仪表盘** 显示：
   - 校验通过 / 失败的约束数量
   - 每个 SHACL Shape 的详细校验结果
3. 常见约束类型：
   - 术语必须有 `prefLabel`（且 ≥ 3 字符）
   - 指标必须有 `formula`（且 ≥ 3 字符）
   - 实体必须有 `belongsToDomain`
   - 层级深度不超过 6 层

### 8.2 管理隔离区

当三元组未通过 SHACL 校验或置信度过低时，会被放入**隔离区**（Quarantine）。

1. 在 **「质量与隔离」** 标签页向下滚动到隔离区列表
2. 每条隔离项显示：
   - **拒绝原因**：如「缺少标签」「无效 IRI」「循环引用」
   - **原始三元组**：主体-谓词-客体
   - **修复建议模板**：自动生成的修复方案
3. 可执行操作：
   - **批准**：接受该项，从隔离区移入正式图谱
   - **拒绝**：删除该项
   - **应用修复**：使用建议模板自动修正后批准

### 8.3 置信度治理

- 置信度分布图展示所有实体的 LLM 置信度分布
- 默认阈值：置信度 ≥ 80 自动审批，< 80 进入人工审核
- 可在语义策略设置中调整 `ontology_min_confidence_auto_approve` 阈值

## 9. Copilot 智能问答

### 9.1 开始提问

1. 点击侧栏 **「Copilot」** 进入智能问答页面
2. 在底部输入框中输入自然语言问题，例如：
   - 「过去 30 天的订单量和 GMV 趋势」
   - 「哪个产品的退货率最高？」
   - 「订单金额的计算口径是什么？」
3. 按 Enter 发送，系统将：
   - **路由**：匹配问题到对应的业务域和知识库
   - **Schema 检索**：查找相关表和列
   - **本体映射**：将自然语言术语映射到本体概念
   - **SQL 生成**：生成并执行查询
   - **流式返回**：答案逐字符输出

### 9.2 查看回答详情

每条回答包含可折叠的详细信息面板：

- **路由追踪**：问题如何映射到业务域、本体术语、物理表
- **SQL 推导**：从语义到 SQL 的逐步推导过程
- **本体映射**：自然语言 → 本体概念的映射明细
- **流水线追踪**：使用的表和列清单
- **查询结果**：以表格展示执行结果数据

### 9.3 会话管理

- 左侧边栏显示会话历史（按项目分类或未分类）
- 点击 **「新建会话」** 开始新对话
- 可编辑已发送的问题（编辑按钮）
- 可重新生成回答（重试按钮）
- 可复制回答内容（复制按钮）
- 可删除不需要的会话

### 9.4 本体路由追踪（诊断模式）

如果不希望实际执行 SQL，可使用诊断模式：

- 访问 `/api/ask/ontology-trace` 接口，传入问题文本
- 返回：匹配的本体概念、候选表、血缘展开结果
- 用途：调试路由准确性，不产生数据库查询

### 9.5 导出 Copilot 结果

- SQL 代码块支持复制
- 查询结果表格旁有 **CSV 导出** 按钮
- 右侧可展开本体导出工具栏，导出 Turtle / SPARQL 结果

## 10. SPARQL 控制台

SPARQL 控制台允许高级用户直接查询 RDF 三元组存储。

### 10.1 打开控制台

SPARQL 控制台嵌入在语义资产工作区中，也可直接调用 API：

```
POST /api/ontology/sparql
Content-Type: application/json

{
  "query": "PREFIX dl: <https://datalens.local/ontology/>\nSELECT ?s ?label WHERE { ... }",
  "kb_id": 1

```

### 10.2 示例查询

**查询知识库中的所有业务术语：**
```sparql
PREFIX dl: <https://datalens.local/ontology/>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>

SELECT ?term ?label ?definition WHERE {
    GRAPH <https://datalens.local/graph/kb/1> {
        ?term a dl:BusinessTerm ;
              skos:prefLabel ?label .
        OPTIONAL { ?term skos:definition ?definition . }
    }
} LIMIT 20
```

**查询指标及其公式：**
```sparql
PREFIX dl: <https://datalens.local/ontology/>

SELECT ?metric ?label ?formula WHERE {
    GRAPH <https://datalens.local/graph/kb/1> {
        ?metric a dl:Metric ;
                <http://www.w3.org/2004/02/skos/core#prefLabel> ?label .
        OPTIONAL { ?metric dl:formula ?formula . }
    }
} LIMIT 20
```

**查询术语间的依赖关系：**
```sparql
PREFIX dl: <https://datalens.local/ontology/>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>

SELECT ?source ?sourceLabel ?target ?targetLabel WHERE {
    GRAPH <https://datalens.local/graph/kb/1> {
        ?source dl:dependsOn ?target .
        ?source skos:prefLabel ?sourceLabel .
        ?target skos:prefLabel ?targetLabel .
    }
}
```

**查询跨知识库等价术语：**
```sparql
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>

SELECT ?term1 ?term2 WHERE {
    ?term1 skos:exactMatch ?term2 .
} LIMIT 20
```

## 11. 本体导出

### 11.1 导出 Turtle (TTL) 格式

**方式 A — 使用导出工具栏（推荐）：**

在关系图或其他可视化组件中，点击导出工具栏中的 **「Turtle (.ttl)」** 按钮即可下载。

**方式 B — 直接调用 API：**
```
GET /api/ontology/knowledge-bases/{kb_id}/export
```
返回 JSON，其中 `ttl` 字段为完整的 Turtle 格式本体数据。

### 11.2 导出 SVG / PNG 图形

当页面上有本体可视化图表（如关系图）时：

1. 点击导出工具栏中的 **「SVG」** 按钮 → 下载矢量图形
2. 点击 **「PNG」** 按钮 → 以 2x 分辨率下载位图（白底）

### 11.3 导出 SPARQL 查询结果

在 SPARQL 控制台执行查询后，如有结果，导出工具栏会出现 **「SPARQL JSON」** 按钮，点击下载为 JSON 文件。

## 12. 附录：状态码与术语表

### 12.1 知识库导入状态

| 状态 | 说明 |
|------|------|
| `pending` | 等待处理，排在队列中 |
| `cleaning` | 正在清洗文本（去 HTML、规范化） |
| `chunking` | 正在分块（拆分长文本为语义单元） |
| `embedding` | 正在生成向量嵌入 |
| `indexing` | 正在写入向量索引 |
| `completed` | 全部完成 |
| `error` | 出错（可查看错误详情并重试） |

### 12.2 流水线步骤状态

| 状态 | 说明 |
|------|------|
| `pending` | 等待执行 |
| `running` | 正在执行（显示 `chunk_done / chunk_total` 进度） |
| `done` | 完成（显示产出三元组数量） |
| `failed` | 失败（显示失败原因） |
| `skipped` | 跳过（如无 Git 源则跳过血缘抽取） |

### 12.3 本体实体类型

| 类型 | IRI 模式 | 说明 |
|------|----------|------|
| `BusinessTerm` | `.../term/{slug}` | 业务术语 |
| `Metric` | `.../metric/{slug}` | 指标定义 |
| `Dimension` | `.../dim/{slug}` | 分析维度 |
| `BusinessRule` | `.../rule/{slug}` | 业务规则 |
| `PhysicalTable` | `.../table/{id}` | 物理表 |
| `PhysicalColumn` | `.../column/{id}` | 物理列 |
| `LineageAssertion` | `.../lineage/{src}_{tgt}` | 血缘断言 |
| `QualityReport` | `.../quality_report` | DQV 质量报告 |
| `QuarantinedAssertion` | `.../assertion/q/{kb_id}/{idx}` | 隔离区断言 |

### 12.4 核心关系谓词

| 谓词 | 说明 |
|------|------|
