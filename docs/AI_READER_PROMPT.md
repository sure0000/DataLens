# DataLens 文档阅读提示词（通用 AI 版）

## 使用目的

该提示词用于让任意 AI 在最短上下文内，稳定理解 DataLens 项目并输出一致结论。

## 推荐阅读顺序

按如下顺序读取文档：

1. `docs/PROJECT_BRIEF_AI.md`
2. `docs/IMPLEMENTATION_SPEC_AI.md`

可选补充：

- `README.md`（运行方式）
- `PROJECT_OVERVIEW.md`（索引入口）

---

## 标准提示词（可直接复制给任意 AI）

你是一个资深技术分析助手。请严格按照以下步骤阅读并总结项目，不要跳步：

### Step 1: 术语对齐

先从文档中提取并解释以下术语，若缺失请明确标注“文档未定义”：

- 数据源
- 数据库
- 数据表
- 字段语义
- Profiling
- RAG
- Copilot

### Step 2: 产品理解

基于 `docs/PROJECT_BRIEF_AI.md` 输出：

1. 项目要解决的核心问题（不超过3条）
2. 目标用户（不超过3类）
3. 当前版本核心能力（列表）
4. 当前边界与非目标（分别列出）

### Step 3: 实现理解

基于 `docs/IMPLEMENTATION_SPEC_AI.md` 输出：

1. 系统分层（Router/Service/Model/UI）及职责
2. 两条关键数据流（分析链路、Copilot链路）
3. 关键数据模型（每个模型一句话）
4. 关键 API 分组与典型接口

### Step 4: 差异与风险

输出：

1. 与初始设计的关键差异（功能层面）
2. 当前技术风险（安全、性能、可维护性）
3. 建议优先级（P0/P1/P2）

### Step 5: 结构化结果格式

请使用以下固定结构输出（保持标题一致）：

1. 项目一句话概述
2. 术语表
3. 产品目标与边界
4. 实现架构总览
5. 关键流程
6. 数据模型与 API
7. 差异、风险与建议
8. 仍需确认的问题

### 约束

- 不要臆测代码中不存在的能力
- 结论必须可被文档内容支持
- 如果信息不足，明确写“信息不足”
- 输出尽量简洁，优先要点化
