## 2. 当前工程中已有的承载能力

| 能力 | 数据模型 | 进入 Copilot 的路径 |
|------|----------|---------------------|
| 列语义 | `ColumnMeta` | `context_builder.build_priority_context` → schema |

## 3. 知识库清洗与结构化（已实现）

### 3.1 文档流水线

```
extract → clean → chunk → embed → structuring → indexed
```

| Stage | 模块 | 产出 |
|-------|------|------|
| clean | `document_cleaner.py` | 去噪声、标点归一化、块质量分 |
| chunk | `document_chunker.py` | `DocumentChunk` |
| embed | `knowledge_pipeline_service.py` | 向量 + tsvector |
| structuring | `chunk_semantic_structuring.py` | `semantic_meta`、entry role、relations 同步 |

### 3.2 语义提取流水线（9 步）

触发：文件上传 / Git 同步 / `POST .../modeling/runs`

| Step | 产出 |
|------|------|
| 术语提取 | `dl:BusinessTerm` + `skos:prefLabel/definition` |
| 指标提取 | `dl:Metric` + `dl:formula/caliber/computedFromTable` |
| 维度提取 | `dl:Dimension` + `dl:dimensionType` |
| 规则提取 | `dl:BusinessRule` + `dl:ruleExpression/ruleType` |
| 关系提取 | `dl:dependsOn/derivedFrom/skos:related` 语义边 |
| 层级构建 | `skos:broader/narrower` 层级树 |
| 血缘提取 | `dl:LineageAssertion` + `dl:transformsFrom`（Git 源） |
| JOIN 提取 | `dl:JoinRelation` + `dl:joinableWith`（Git 源） |
| 领域术语提取 | 跨领域术语对齐 |

全部产出经 `clean_triples()` 9 阶段清洗 → SHACL 校验 → production / quarantine。
