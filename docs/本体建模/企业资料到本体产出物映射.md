# 企业资料 → 本体七类产出物映射

> 对照 [本体建模理论标准](./本体建模理论标准.md) 定义的七类标准产出物，梳理企业各类型原始资料经清洗后可产出的本体要素。作为 DataLens 清洗管线设计的参照基线。

---

## 一、七类标准产出物速查

| 编号 | 产出物 | 技术基座 | 核心内容 |
|------|--------|---------|---------|
| ① | 概念词典 / 词汇表 | SKOS | 术语名、类型枚举、业务定义、同义词、上位概念、来源锚点 |
| ② | 类层次结构 | RDFS + OWL | `rdfs:subClassOf` 树、`owl:disjointWith`、`owl:equivalentClass` |
| ③ | 关系定义 | OWL ObjectProperty | domain/range、Transitive/Symmetric/Functional/InverseFunctional、属性链公理 |
| ④ | 属性定义 | OWL DatatypeProperty | 属性名、domain 类、range 数据类型（xsd 映射） |
| ⑤a | OWL 推理公理 | OWL Axioms | `someValuesFrom`、`allValuesFrom`、`FunctionalProperty`、`propertyChainAxiom` |
| ⑤b | SHACL 校验约束 | SHACL Shapes | `sh:minCount`、`sh:class`、`sh:in`、`sh:pattern` |
| ⑤c | SWRL / Datalog 规则 | SWRL / 自定义规则 | IF-THEN 推导规则、口径判定逻辑 |
| ⑥ | 实例数据（ABox） | RDF + OWL ABox | `rdf:type` 声明、ObjectProperty 断言、DatatypeProperty 字面量 |
| ⑦ | 对齐与可视化 | SKOS mapping + 前端 | `skos:exactMatch`/`closeMatch`、交互式图谱 |

### 2.1 文档类资料

#### 数据字典 / 业务 Wiki

| 产出物 | 覆盖度 | 具体内容 | 清洗方式 |
|--------|--------|---------|---------|
| ① 概念词典 | ████ 核心 | 术语名称、业务定义、同义词（altLabel）、术语类型枚举（metric/enum/time/dimension） | LLM 从文档 chunk 提取 term/definition/synonyms/type |
| ② 类层次 | ██ 辅助 | 术语归属的上位概念（parent_concept → `skos:broader`） | LLM 提取 parent_concept 字段 |

### 2.2 数据库元数据类资料

#### 数据库 DDL

| 产出物 | 覆盖度 | 具体内容 | 清洗方式 |
|--------|--------|---------|---------|
| ④ 属性定义 | ████████ 核心 | 列名 → DatatypeProperty 名称，列类型 → xsd 类型映射（VARCHAR → xsd:string，INT → xsd:integer） | Schema 采集 JDBC → ColumnMeta |
| ⑥ ABox 实例 | ████████ 核心 | 每张表 → `dl:PhysicalTable` 个体，每列 → `dl:PhysicalColumn` 个体 | `schema_extractor.py` 自动采集 |
| ⑤a OWL 公理 | ███ 重要 | PRIMARY KEY → `owl:FunctionalProperty`；NOT NULL → `owl:someValuesFrom` 存在约束；FOREIGN KEY → `rdfs:domain`/`rdfs:range` | ❌ 当前未实现自动转换 |
| ⑤b SHACL 约束 | ██ 辅助 | CHECK 约束 → `sh:minInclusive`/`sh:maxInclusive`/`sh:pattern`；枚举类型 → `sh:in` | ❌ 当前未实现自动转换 |
| ③ 关系定义 | ██ 辅助 | 外键 → ObjectProperty 候选（如 `orders.customer_id → Customer`） | 需结合 JOIN 分析 |

DataLens 实现：✅ schema_extractor（822 行）；❌ DDL 约束 → OWL 公理自动转换

## 五、优先级建议

| 优先级 | 缺口 | 理由 |
|--------|------|------|
| **P0** | DDL 约束 → ⑤a OWL 公理 | DDL 是数据源接入的第一手资料，NOT NULL/UNIQUE/FK 等信息已有但被丢弃 |
| **P1** | 代码表 → ⑤b SHACL 约束 | 枚举校验是最常见的数据质量问题，且实现成本低 |
| **P1** | SQL/dbt → AST 级血缘 | 提升血缘提取的准确率和覆盖率，减少 LLM 幻觉 |
| **P2** | 行业标准 → ⑦ 对齐 | 提升 Copilot 回答的专业性和跨企业互操作能力 |
| **P2** | KPI/制度 → ⑤c SWRL 规则 | 将口Judge判定逻辑系统化，但 ROI 取决于规则复用率 |
| **P3** | MDM / 建模工具导入 | 连接器开发成本高，按需实施 |
